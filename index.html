    最近讀到 LogRocket 的 What should a modern CSS boilplate look like? 說到 CSS reset 的觀念，隨著時代演進，不該在只像過往一樣把所有瀏覽器預設的 CSS 屬性清乾淨，再重新加上去，可以思考一下其中的屬性是不是有保留的價值，文章很好，但就是...沒有範例，只給了程式碼，所以我做了這個網頁呈現文章中所描述的狀況，或是提供我找到的範例，期待可以作為這些條目的閱讀、操作，和理解的輔助😁。

    Don't reset margins

    瀏覽器合理地預設了一些元素的邊界值，這篇文章中的 header 和 p tags 都沒有重設過邊界值，如果把它們 reset 掉的話，呈現出來如下方的灰底區；段落間的行距沒有了：

    Definitely reset the box model

    因為 border-box 是比較後面才出來的，為了不影響過去的網站設定，它不是瀏覽器預設的 box model 設定，如果我們不主動做 box-sizing: border-box，對於有 padding 和 margin 的元素，就很容易超出預想的元素大小：
